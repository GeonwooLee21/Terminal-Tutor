# Terminal Tutor

> 터미널이 낯선 사용자가 번호 입력만으로 터미널 명령어를 배우고 익힐 수 있게 돕는 CLI 튜터

---

## 프로젝트 소개

우리는 컴퓨터를 실행하면 가장 먼저 바탕화면을 마주하고, 마우스 클릭으로 원하는 것을 조작한다. 이는 터미널에서 ls와 cd를 차례대로 입력하는 것과 동일한 행동이다. 그러나 전자의 GUI에 익숙해진 현대인에게 후자의 CLI인 터미널은 낯설고 어려운 도구로 느껴진다.

하지만 컴퓨터를 사용하다 보면 예기치 못하게 터미널을 켜야 하는 순간이 찾아온다. 실제로 컴퓨팅사고와 SW코딩 중간고사 실습 평가 때 학생들은 Jupyter Notebook을 터미널로 실행해야만 했다. 이때 꽤 많은 학생들이 터미널 조작에 어려움을 겪었는데, 이러한 모습이 해당 프로젝트의 강력한 동기로 작용했다.

기존의 TUI 파일 매니저들은 Vim 키바인딩 기반이라 오히려 초보자에게 더 낯선 경험을 제공한다. 일반 사용자에게 터미널은 리모컨 하나 없이 마주하는 하얀/검은 화면에 불과하다. **OS가 컴퓨터 내부 구조를 모르는 사용자도 컴퓨터를 쉽게 사용할 수 있도록 돕듯, Terminal Tutor는 리모컨처럼 버튼 선택만으로 터미널을 조작할 수 있는 인터페이스를 제공한다. 터미널이 낯선 초보자부터 어쩔 수 없이 터미널을 켜야 하는 일반 사용자까지, 터미널 진입장벽을 낮추는 것을 목표로 한다.**

---

## 핵심 기능

- **키패드 기반 명령어 실행** — `ls`, `cd`, `mkdir` 등 주요 명령어를 번호 입력만으로 선택하고 실행한다.
- **현재 위치 시각화** — 사용자가 터미널에서 현재 어느 디렉토리에 위치하는지를 경로 형태로 직관적으로 표시한다.
- **위험 명령어 실행 전 경고** — `rm` 등 복구가 어려운 명령어 실행 시, 사전 경고 메시지를 띄워 초보자의 실수를 방지한다.

---

## 지원 명령어

터미널 입문자가 가장 먼저 익혀야 할 기초 명령어 9가지를 선정해 제공한다.

| 번호 | 명령어 | 설명 | 분류 |
|------|--------|------|------|
| 1 | `ls` | 현재 디렉토리의 파일·폴더 목록 출력 | 탐색 |
| 2 | `pwd` | 현재 디렉토리 경로 출력 | 탐색 |
| 3 | `cd` | 디렉토리 이동 | 탐색 |
| 4 | `mkdir` | 새 폴더 생성 | 생성 |
| 5 | `touch` | 새 파일 생성 | 생성 |
| 6 | `cat` | 파일 내용 출력 | 확인 |
| 7 | `clear` | 터미널 화면 초기화 | 확인 |
| 8 | `rm` | 파일 삭제 ⚠️ | 삭제 |
| 9 | `rm -r` | 폴더(및 하위 항목) 삭제 ⚠️ | 삭제 |

## 실행 방법

**요구 사항**
- Python 3.10 이상
- macOS / Linux (터미널 환경)

**1. 저장소 클론**

```bash
git clone https://github.com/GeonwooLee21/Terminal-Tutor.git
cd Terminal-Tutor
```

**2. 라이브러리 설치**

```bash
pip install rich
```

> 설치 중 오류가 발생하면 가상 환경을 사용해 설치한다.
> ```bash
> python3 -m venv venv
> source venv/bin/activate
> pip install rich
> ```

**3. 실행**

```bash
python main.py
```

**4. 조작 방법**

**온보딩 흐름**

프로그램 실행 시 두 단계의 질문을 통해 모드가 결정된다.

```
터미널을 처음 사용해 보시나요? [y/n]
├── y → beginner 모드로 진입
└── n → 그래도 한번 둘러보시겠습니까? [y/n]
    ├── y → normal 모드로 진입
    └── n → 삭제 안내 메시지 출력 후 프로그램 종료
```

> `n → n` 선택 시 프로그램이 자동으로 삭제되지는 않는다.  
> 사용자가 직접 삭제할 수 있도록 삭제 명령어(`rm -rf Terminal-Tutor/`)를 안내한 뒤 종료된다.

**키패드 조작**

| 입력 | 동작 |
|------|------|
| 버튼 번호 (예: `1`) | 해당 명령어 실행 |
| `b` | 인자 선택 화면에서 키패드로 돌아가기 |
| `0` | (beginner 모드) 명령어 이름 표시 / 숨김 토글 |
| `g` | (beginner 모드) 졸업 메시지 출력 후 종료 |
| `q` | 프로그램 종료 |

## 사용된 Python 개념

| 개념 | 활용 |
|------|------|
| 딕셔너리, 리스트, 함수 | 명령어 버튼 구조 및 키패드 로직 설계 |
| `subprocess` 모듈 | Python 코드 안에서 외부 프로세스(터미널 명령어 등)를 실행하고 그 결과를 받아온다. 사용자가 선택한 명령어를 실제 터미널에 실행하는 데 활용한다. |
| `rich` 라이브러리 | 터미널 안에 텍스트 기반 UI(버튼, 색상, 레이아웃 등)를 렌더링할 수 있게 해주는 라이브러리다. 키패드 화면 구성 및 위험 명령어 빨간색 강조에 활용한다. |
| 클래스 (OOP) | `Button` 클래스로 명령어의 이름·설명·위험 여부 등 속성과 `execute()` 메서드를 하나로 묶어 관리한다. |
| `os` 모듈 | 운영체제와 상호작용하는 기능을 제공한다. 현재 디렉토리 경로 탐색 및 파일·폴더 목록 조회에 활용한다. |
| 조건문 | 위험 명령어(`rm` 등) 실행 여부를 감지하고, 해당 명령어가 선택됐을 때 경고 메시지를 출력해 사용자의 최종 확인을 받는 데 활용한다. |

## AI 도구 사용 내역

**사용 도구**: Claude (Anthropic)

---

**1. 라이브러리 선택**
- **어디에 사용**: 프로젝트 구현에 필요한 라이브러리 선정
- **받은 도움**: 터미널 명령어 실행에 `subprocess`, 디렉토리 탐색에 `os`, 터미널 UI 렌더링에 `rich`가 적절하다는 판단을 제안받음
- **내 판단**: 각 라이브러리의 역할을 이해한 뒤 채택

---

**2. 라이브러리 사용법 학습**
- **어디에 사용**: `subprocess.run()`, `os.chdir()` / `os.listdir()` / `os.path.join()`, `rich`의 `Console` / `Table` 사용법
- **받은 도움**: 각 함수의 매개변수와 반환값, 사용 예시를 학습함
- **내 판단**: 공식 문서와 대조하며 확인 후 적용

---

**3. `get_arg_buttons()` 리스트 컴프리헨션 리팩토링**
- **어디에 사용**: `get_arg_buttons()` 함수 코드 개선
- **받은 도움**: `for`문 + `append()` 형식으로 작성한 폴더 필터링 로직과 버튼 생성 로직을 더 간결하게 표현할 수 있는 리스트 컴프리헨션 방식을 제안받음
- **내 판단**: 두 곳의 리스트 컴프리헨션 문법을 검토 후 이해함. 조건 필터링이 포함된 `[e for e in entries if ...]` 형태와 객체 생성을 담은 `[Button(...) for t in targets]` 형태가 각각 어떻게 동작하는지 확인 후 채택 (후자의 리스트 컴프리헨션 구문은 이후 버튼 생성 로직에 `.`, `..` label 분기가 추가되면서 리스트 컴프리헨션으로 표현하기 어려워져 일반 `for`문 + `append()` 형식으로 재작성함)

---

**4. `show_keypad()` 리스트 언패킹 (`*row`)**
- **어디에 사용**: `show_keypad()`에서 `table.add_row()`에 리스트를 넘기는 방법
- **받은 도움**: 리스트를 `*`로 언패킹해서 함수 인자로 전달하는 방법 학습
- **내 판단**: `*row`가 `row[0], row[1], row[2]`를 풀어서 넘기는 것과 동일하다는 걸 이해함. 동작 확인 후 채택

---

**5. `ARG_PROMPTS` 딕셔너리 설계**
- **어디에 사용**: `mkdir` / `touch` 분기 처리 설계
- **받은 도움**: 명령어별 안내 문구를 분기로 나누어 처리하지 말고, `ARG_PROMPTS` 딕셔너리를 이용해 분기를 나누지 않는 방식을 제안받음
- **내 판단**: 각각 `elif`로 분기하는 것보다 코드 중복이 줄어들고, 명령어가 늘어나도 딕셔너리 항목만 추가하면 된다는 걸 이해함. 동작 확인 후 채택

---

**6. ANSI 이스케이프 코드로 스크롤 버퍼 초기화**
- **어디에 사용**: `onboarding()` 및 `graduation()` 함수에서 화면 초기화
- **받은 도움**: macOS 터미널에서 `clear` 명령어는 화면을 밀어올릴 뿐 스크롤 버퍼를 지우지 않는다는 사실을 확인함. 이후 AI를 통해 버퍼까지 지우려면 ANSI 이스케이프 코드(`\033[2J\033[3J\033[H`)를 직접 출력해야 한다는 것을 학습함
- **내 판단**: 동작 확인 후 채택

---

**7. 예외 처리 대상 검토**
- **어디에 사용**: 터미널 명령어를 다루는 프로그램 특성상 예외 상황이 많아 혼자 힘으론 모든 케이스를 찾기 어렵다고 판단해, 현재 코드에서 예외 처리가 필요한 곳을 분석 요청
- **받은 도움**: `get_arg_buttons()`의 `PermissionError`, `cd` 블록의 `FileNotFoundError`·`PermissionError`·`NotADirectoryError`, `mkdir`/`touch` 빈 입력값 총 5가지 제안받음
- **내 판단**: 실제 코드 로직과 대조해 5가지 중 2가지만 채택해 적용함. `get_arg_buttons()`의 `PermissionError`는 현재 디렉토리에 이미 위치해 있다는 것 자체가 접근 권한이 있다는 의미이므로 불필요. `cd`의 `FileNotFoundError`·`NotADirectoryError`는 현재 디렉토리의 항목을 번호로 매핑해 그 범위 내에서만 선택 가능한 구조상 발생 불가. `PermissionError`(in `cd`)와 `mkdir`/`touch`시 입력값이 빈 경우만 처리하면 충분함을 확인

## 한계와 다음 단계

**현재 한계**
- 본 프로그램을 실행하려면 터미널에서 직접 `python main.py`를 입력해야 하므로, 터미널을 처음 접하는 사용자에게는 실행 자체가 진입장벽이 될 수 있음
- 본 프로그램으로 터미널에 '익숙'해지는 것은 가능하지만, 터미널 명령어를 '학습'한다고 느끼게 하기엔 한계가 있음
- 번호 선택 방식으로만 명령어를 실행하고, 명령어의 종류와 수는 고정되어 있으므로, 사용자가 명령어를 학습하는 경험이 제한적임
- `rich` 기반의 UI를 CLI 환경에 렌더링하는 형태라, UI 출력이 매끄럽지 않을 때가 많음
- macOS / Linux 전용으로, Windows 환경에서는 동작하지 않음 (`ls`, `rm` 등 Unix 계열 명령어 사용)

**다음 단계 (NICE / LATER 기능)**
- 터미널을 처음 켠 사용자가 특정 문구(예: `I want a tutor`)를 입력하면 자동으로 Terminal Tutor가 실행되도록 해 접근성 개선
- 사용자 이름·사용 기록·졸업 여부를 JSON으로 저장하는 기능
- 명령어 수행 횟수에 따라 경험치가 쌓이는 레벨 시스템(JSON에 저장된 데이터 활용): 레벨에 따라 제공되는 명령어가 달라지거나, 번호 선택이 아닌 명령어 직접 입력 방식으로 전환되는 등 사용자가 프로그램을 통해 얻는 경험을 단계적으로 설계해 학습 효과 개선
- 터미널 화면을 직접 제어하는 `curses` 기반으로 UI 렌더링 방식 개선
- Windows 지원 (`dir`, `del` 등 명령어 자동 전환)
